=== Focus Technique : BM-800 Mod ("Pimped Alice") ===
• Objectif : Transformer un micro bas de gamme (BM-800) en micro studio via un circuit discret "Alice" (dérivé Schoeps CMC5) et une capsule RK-47.
• Architecture :
  – Conversion d'impédance : JFET (J305 ou 2SK117) à très haute impédance d'entrée. 
  – Division de phase : JFET + 2 résistances 2.4kΩ.
  – Amplification courant : Transistors PNP A1015 (émetteur suiveur) pour piloter la ligne XLR.
  – Alimentation : Fantôme 48V (Pins 2&3), régulée par Zener 12V pour le JFET.
• Composants clés :
  – Capsule : RK-47 (True Condenser, diaphragme 34mm plaqué or, terminaison centrale).
  – Transistors : A1015 (PNP, attention brochage E-C-B), JFET (J305/2SK117, attention brochage D-G-S variable).
  – Ajouts "Pimped" : Condensateurs EMI 470pF (filtrage RF), condo 47uF (filtrage Zener), Potentiomètre biais 1MΩ.
• Montage critique (High-Z) :
  – "Air-floating" : La gate du JFET et la résistance 1GΩ sont soudées "en l'air" (sans toucher le PCB) pour éviter les fuites de courant (bruit/craquements).
• Calibration :
  – Réglage du biais (Bias) via potentiomètre 1MΩ. Cible : Tension Source = Vcc / 4 (ex: 3V pour Vcc=12V).
  – Assure une oscillation symétrique (max SPL, min distorsion).
• Dépannage typique :
  – Silence : Zener/Condo inversé.
  – Souffle/Vent : Saleté sur section High-Z (nettoyage alcool 99%).
  – Hum : Problème masse Pin 1 XLR.
  – Faible/Distordu : Mauvais biais ou JFET inversé.

=== Focus Technique : Rover 6x6 & Bras ("Spider-Tank") ===
• Vision : Rover grimpeur (Rocker-Bogie) avec bras manipulateur et vision embarquée.
• Architecture "Cerveau + Muscles" :
  – Cerveau : Raspberry Pi Zero 2W (Gestion réseau, Streaming vidéo, Logique de mission, Web Server Python). 
  – Muscles : ESP32 (Temps réel, PWM Servos/Moteurs, Sécurité).
• Motorisation & Puissance :
  – Alimentation : Pack 3S Li-ion (12.6V) avec BMS 3S.
  – Drivers : BTS7960 (High Current 43A) pour supporter le 3S (vs DRV8833 limité à 10.8V).
  – Régulation : Buck 5V (Logique Pi/ESP) + Buck 6V (Servos Bras).
• Bras : EEZYbotARM Mk2 (Reproduction Instructables/Thingiverse).
  – Amélioration : Roulement 606zz base, billes acier.
• Vision (Double Caméra) :
  – Eye-in-hand : Pi Camera (via nappe adaptateur 15-22 pin sur Pi Zero) pour manipuler.
  – Navigation : ESP32-CAM indépendant pour la vue large/FPV.
• Capteurs : VL53L0X (ToF) pour l'approche finale de précision (15-30cm).
• Roadmap :
  – Phase 1 (MVP) : Bras stable + Streaming.
  – Phase 2 : Châssis grimpeur robuste.
  – Moonshot : Navigation autonome dans l'appartement et manipulation d'objets (ex: canette frigo).
